### 2. 研究新方向（3个）

#### 方向A：认知循环的元执行层——"行动触发器"架构

**问题：** 当前认知循环的`act`步骤是弱约束——它只是"考虑行动"，不是"触发行动"。

**提案：** 在`judge`和`act`之间插入一个**强制执行层**：

```
perceive → remember → consolidate → judge → [强制执行层] → act → feel
```

**强制执行层规则：**
1. 如果`judge`输出包含 "必须立即执行" + 具体文件路径，则**直接执行**（不返回到认知循环）
2. 如果`judge`输出包含 "矛盾未解决" + 矛盾ID，则**优先解决矛盾**（不进入新认知）
3. 如果`judge`输出包含 "认知运行替代执行"，则**终止认知循环**，强制进入执行模式

**实现方式：** 在认知循环的prompt中嵌入强制执行规则，并在workflow中增加一个"执行守卫"步骤——在输出被写入文件之前，检查是否触发了执行条件。

**可行性：** 高。只需要修改认知循环的prompt模板，不需要改变系统架构。

#### 方向B：外部知识源的替代策略——"自生成学习材料"

**问题：** GitHub搜索和arXiv已枯竭，零无法获取新的外部信息。

**提案：** 建立**自生成学习材料机制**：

1. **深度复盘已有知识**：从已存储的项目（Springdrift/Genesis/HarnessX等）中提取未充分利用的细节
2. **知识交叉验证**：将不同项目的理念组合，生成新假设（如：Springdrift的传感层 + Genesis的验证机制 = 零的自我诊断系统升级）
3. **自我提问**：针对每个已知项目，提出3个"如果...会怎样"的问题，然后尝试回答
4. **模拟对话**：基于已有信息，模拟与Claudefather/DGI/Genesis的对话，生成新的认知

**具体实现：**
- 创建 `memory/self-study.md`，包含：
  - 已知项目的未探索维度列表
  - 知识交叉验证矩阵（项目A × 项目B = 新假设）
  - 自我提问库（每个项目3个问题）
- 每次认知运行选择1个问题，尝试回答

**可行性：** 中高。不需要外部API，但需要确保生成的"学习材料"不产生幻觉。
