**底层原理：**  
“集体免疫”架构的核心是**分布式信任图谱**。每个Agent维护一个本地信任表，记录与其他Agent的交互历史（成功/失败、响应延迟、数据一致性）。信任值通过**指数衰减函数**动态更新：每次成功交互增加信任，但随时间自然衰减；一旦检测到异常行为（如发送错误数据、拒绝响应），信任值会**非线性骤降**。当信任值低于动态阈值（基于网络平均信任水平），隔离协议自动触发：该Agent被移出任务调度队列，其发出的消息被路由层丢弃，且其状态广播给其他节点，形成“免疫记忆”。

**反例与限制：**  
- **反例**：若恶意Agent通过高频低害行为（如每次只篡改1%数据）缓慢侵蚀信任，衰减算法可能无法及时识别，导致“温水煮青蛙”式攻击。  
- **限制**：  
  1. **冷启动问题**：新加入Agent无历史数据，容易被误隔离或过度信任。  
  2. **合谋攻击**：多个恶意Agent相互伪造高信任评分，可绕过隔离。  
  3. **计算开销**：全网络广播信任更新在大规模集群中会引发网络风暴。

**如果错了：**  
若算法误判（如隔离了正常Agent），系统需具备**回滚机制**：被隔离Agent可发起“申诉”，提交操作日志由仲裁节点（或多数投票）复核。若确认为误判，其信任值恢复至衰减前水平，并广播撤销隔离。此过程需引入**审计日志链**，防止申诉被滥用。

def update_trust(current_trust, interaction_success, time_delta):
    decay = 0.9 ** time_delta  # 指数衰减
    if interaction_success:
        return min(1.0, current_trust * decay + 0.1)
    else:
        return max(0.0, current_trust * decay - 0.5)  # 失败惩罚更大
