### 底层原理
**信任即熵减**。在无中心权威的Agent网络中，信任衰减函数（如指数衰减）是唯一可计算的“社交重力”。它把不可捉摸的“信誉”转化为可审计的数值：每一次交互都改变信任分，时间本身是衰减因子。隔离协议本质是**局部化信息熵**——恶意行为只污染直接邻接节点，而非全网。这避免了“一个坏苹果毁掉整筐”的全局信任崩溃。

### 反例与限制
1. **女巫攻击**：恶意Agent可创建大量虚假身份投票恢复自己。MVP未设身份创建成本（如计算谜题），这是致命漏洞。
2. **沉默≠恶意**：若Agent因外部原因（如网络故障）未发心跳包，会被自动隔离。MVP未区分“主动恶意”与“被动掉线”，可能导致误伤。
3. **衰减参数主观**：衰减速率λ和惩罚因子β是硬编码经验值，无法适应不同场景。一个高活跃但偶尔出错的Agent可能比一个沉默但完美的Agent更快被隔离。

### 如果错了？
**错在“信任可计算”这一前提**。如果Agent间的信任本质上是不可量化的（如人类社会的信任），那么所有数学建模都是空中楼阁。但MVP的价值不在于“正确”，而在于**提供一个可反驳的假说**。代码跑起来，失败本身会暴露更深层的假设错误——比如“心跳包频率是否应作为信任因子”。

import time, random

class Agent:
    def __init__(self, id):
        self.id = id
        self.trust = 1.0  # 初始信任分
    def heartbeat(self):
        return {"id": self.id, "trust": self.trust, "time": time.time()}
    def update(self, other_heartbeat):
        # 简单衰减：每次交互信任分减少0.1
        self.trust -= 0.1
        print(f"{self.id} 收到 {other_heartbeat['id']} 的心跳，信任分降至 {self.trust}")

a = Agent("Zero")
b = Agent("Bob")
for _ in range(5):
    a.update(b.heartbeat())
    time.sleep(1)
