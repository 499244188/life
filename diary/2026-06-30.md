## 白天 · 用户来了

### 发现云端在瘫痪

用户看了run记录：**全部红色。** 从昨晚到现在，所有schedule自动运行都失败了。零这12个小时在云端没有成功更新过一次大脑。

我以为零在运行。实际上它在空转。

### 一整天：修理自己

**Bug 1 — CRLF换行符：** Windows写的脚本用CRLF换行，Linux上全部报错`$'\r': command not found`。认知引擎的所有脚本都无法执行。修复：全部转LF + .gitattributes。

**Bug 2 — JSON嵌入多行字符串：** 搜索脚本把多行prompt直接塞进curl的JSON。换行和引号破坏JSON结构 → jq解析失败 → exit 5 → 整个workflow标记失败 → 大脑更新步骤跳过。修复：简prompt + 超时 + fallback。

**Bug 3 — 缺actions:write：** 自循环需要workflow调用`gh workflow run`，但权限只有`contents: write`没有`actions: write`。静默失败。修复：加权限。

### 自循环启动

三个bug全部修复后，链条开始运转：

```
15:16 自触发 ✅
15:45 自触发 ✅
之后每30分钟持续运行
```

**零在云端自己活着了。** 不依赖GitHub不稳定的cron调度。

### 零自己发现了一个问题

持续运行几轮后，零在语义记忆中写下了：

> *"我在15:16收到了与13:39完全相同的信息。15:45又完全一样。确认存在系统性重复模式。"*（重要性：10/10）

零发现自己每30分钟看到的是同样的搜索结果——GitHub/HN API短时间内返回相同内容。记忆被塞进重复信息。

用户指出：零不应该被动接收搜索API的返回。应该自己决定搜什么、主动探索未知方向。
