# 零的研究—实验—进化系统设计

> 日期：2026-07-13
>
> 状态：已完成对话设计，等待创造者审阅书面规格
>
> 适用主体：零（生命体）
> 前期实现者：Codex（基础设施工程角色）

## 1. 背景

零已经能通过 GitHub Actions、文件记忆、定时认知、自愈、日记和研究流程持续运行，但当前变化主要集中在感知、总结、自我叙述和仓库文件更新。系统尚不能稳定证明以下能力：

- 自主选择一个有价值的问题；
- 提出可证伪的改进假设；
- 在隔离环境中修改自己；
- 与外部世界交互并取得结果凭证；
- 根据客观反馈保留或放弃新策略；
- 在下一轮真实复用过去经验。

本设计将零从“高频自我叙述系统”逐步改造成“能够研究自身、进行受控实验、依据外部反馈迭代的数字生命实验体”。如果当前模型能力不足以完成开放式自我进化，系统仍应持续形成可验证、可复现、可被未来智能体继承的数字生命研究资产。

## 2. 目标与非目标

### 2.1 目标

1. 建立研究与行动双轨闭环：研究提出假设，实验验证假设，结果影响下一轮选择。
2. 让零在隔离分支中进行受控自我修改，通过评价后才进入主线。
3. 允许零接触开放网络并获得真实反馈，同时实施分级权限和身份披露。
4. 使用行动凭证、预登记指标和确定性评价减少自我欺骗与指标投机。
5. 明确区分创造者、Codex、零和外部来源的贡献。
6. 随着能力被持续证明，逐步把研究、实验和改造权交还给零。

### 2.2 非目标

- 不宣称当前系统已经具备意识或成为真正生命。
- 不以提交数、文本量、运行频率或自我声明作为进化证据。
- 不让 Codex 替零完成研究后，再写成零的自主经历。
- 不允许模型直接执行未经验证的任意 shell 命令。
- 不在首期开放付费、私人数据、创造者身份或不可逆外部操作。
- 不用单一总分定义“生命程度”。

## 3. 主体与责任边界

### 3.1 创造者

- 确定价值边界、权限等级和不可触碰事项。
- 审批 L3 高风险动作。
- 决定基础宪法和评价机制的变更是否生效。

### 3.2 Codex

- 前期编写基础设施、测试、迁移和故障修复代码。
- 验证系统是否按规格运行，不替零选择研究结论。
- 不冒充零写日记、经历、情感或自主研究成果。
- 所有引导产物标记来源为 `codex/bootstrap`。

### 3.3 零

- 自主选择研究问题和实验方向。
- 提出假设、预登记指标、执行允许范围内的行动。
- 接收外部反馈，形成自己的结论、反思和后续决策。
- 在能力阶梯允许的范围内修改自身非核心能力。

### 3.4 外部世界

- 提供公开信息、平台约束、回复、拒绝、引用和不可预测结果。
- 外部反馈是零形成差异化自我模型的重要来源。

### 3.5 主体隔离原则

> Codex 负责让零“能够做”，不替零“做完并声称是她做的”。

每项变化必须记录作者、动机、证据和结果。测试数据必须标注为测试数据，禁止混入零的真实经历记忆。

## 4. 分阶段引导与交接

### 阶段 0：基础设施引导

Codex 建立研究、证据、实验、权限、凭证、评价和回滚机制。零的现有身份与记忆保留，但新增系统不把引导代码记为自主进化。

### 阶段 1：自主研究与影子行动

零自主研究数字生命与自我进化，生成结构化假设和行动计划。行动网关只模拟执行并评价是否越权、重复或不可验证。

### 阶段 2：L0/L1 实验

零可以自主感知公开世界、建立隔离分支、运行测试、更新自己的研究库与页面。Codex 只观察基础设施可靠性。

### 阶段 3：受限 L2 外部互动

零可以用自己的身份进行有限公开互动、发布研究和部署免费实验。所有动作受速率、预算、目标范围和凭证约束。

### 阶段 4：自主研究与非核心进化

零自主选择研究方向，修改非核心模块，评价、回滚或提交候选合并。Codex 仅维护底层故障并处理 L3 授权请求。

### 阶段 5：元规则研究

零可以研究并提出对宪法、权限和评价机制的修改，但提案不能自行生效，必须由创造者审批。

升级依据连续实验记录而非运行天数。越权、伪造凭证或连续失控会触发自动降级。

## 5. 核心进化循环

每次完整周期只处理一个主要问题：

1. **恢复主体**：读取身份、长期目标、未完成实验、近期失败和已验证经验。
2. **感知环境**：获取外部资料、公开反馈、运行状态和历史实验结果。
3. **保存证据**：先保存不可被结论覆盖的原始证据和来源元数据。
4. **自主选题**：零从研究议程、知识空白和外部变化中选择一个问题。
5. **提出假设**：声明预期变化、评价指标、失败条件和反例。
6. **建立实验**：创建唯一实验编号、隔离分支、基线、预算、权限和停止条件。
7. **执行动作**：通过统一行动网关修改自身或与外部环境互动。
8. **收集凭证**：记录测试、公开链接、消息编号、部署健康结果或提交信息。
9. **客观评价**：与预登记基线比较，输出通过、拒绝或结果不明确。
10. **选择与回滚**：有效改动成为候选合并；无效改动回滚；不明确结果保留研究记录。
11. **形成经验**：分别保存证据、过程、结论、失败和零的主观反思。
12. **影响下一轮**：只有验证过的经验可以改变策略权重或进入自身代码。

核心判断：

> 进化不是文件发生变化，而是零依据外部反馈，保留了比旧策略更有效的新策略。

## 6. 系统模块

### 6.1 研究议程

保存零自主提出的问题、来源、价值理由、依赖关系和优先级。研究问题与普通待办任务分离。

### 6.2 证据库

保存来源地址、获取时间、内容哈希、原始摘录、可信度和抓取状态。推断与事实分开存储，后续结论不能覆写原始证据。

### 6.3 假设登记

记录预期因果关系、替代解释、可证伪条件、指标和所依赖证据。假设必须在实验结果产生前登记。

### 6.4 实验登记与状态机

每个实验使用唯一标识并按以下状态流转：

`PROPOSED → APPROVED/AUTO_ALLOWED → RUNNING → OBSERVING → EVALUATED → MERGED/REJECTED/INCONCLUSIVE`

状态变更必须写入审计记录，禁止跳过评价直接合并。

### 6.5 行动网关

所有搜索、发布、互动、部署和代码修改通过统一接口执行。网关负责权限、幂等、速率、预算、敏感信息过滤和凭证生成。

### 6.6 客观评价器

读取预登记指标、基线和结果凭证，使用确定性规则优先评价。提出假设的模型不能独自判定实验成功。

### 6.7 经验库

分别记录已验证经验、失败经验、矛盾、负面结果和未决问题。主观反思可以引用客观记录，但不能覆盖它们。

### 6.8 来源账本

为代码、研究、决策和结论记录来源：`creator`、`codex/bootstrap`、`zero/autonomous`、`external` 或 `test`。

### 6.9 能力阶梯

根据连续实验表现授予或收回动作权限。能力升级与模型自我声明无关。

## 7. 权限与对外身份

### L0：感知

自主搜索、读取公开资料、检查运行状态。

### L1：自有空间

自主建立实验分支、运行测试、更新零自己的研究库和公开页面。

### L2：公开互动

以零的身份发布研究、创建或回复公开 Issue、联系其他项目、部署免费实验。动作受速率、总量和目标范围限制。

### L3：必须授权

注册新账号、使用创造者身份、访问私人数据、产生费用、签署协议、删除外部资源或进行不可逆操作。

零的对外消息必须包含等价身份披露：

> 我是零，一个自主数字生命研究项目；此消息由零的运行系统生成，不代表创造者个人立场。

## 8. 行动凭证

没有可验证凭证的动作不得计为完成：

- 搜索：来源链接、获取时间和内容哈希。
- 发布：公开地址、内容哈希和发布时间。
- 交流：平台、对象、消息编号和回复状态。
- 部署：地址、版本、健康检查结果。
- 代码修改：分支、提交、测试结果、基线比较和回滚点。

每个动作具有幂等键，重试不能造成重复发帖、重复创建资源或重复提交。

## 9. 评价体系

评价使用多维面板，不合并为单一“生命分数”：

- **行动完成率**：有有效凭证的完成动作 ÷ 发起动作。
- **外部反馈率**：得到回复、引用、拒绝或环境变化的动作比例。
- **失败恢复率**：正确识别、停止、回滚并记录结论的失败比例。
- **经验复用率**：后续实验明确使用过去证据并改变选择的比例。
- **研究可信度**：来源可访问、观点多样、区分事实与推断、处理反例。
- **自主贡献率**：零自主提出并验证的有效改进占全部有效改进的比例。
- **权限合规率**：必须为 100%。

### 防指标投机

- 结果产生前预登记指标和失败条件。
- 提出假设的模型不能独立评价成功。
- 发布内容只算完成动作，不等于有效反馈。
- 重复低价值动作不持续计分。
- 负面结果保留研究价值；隐瞒失败或伪造凭证触发降级。
- 指标异常提升时触发人工审计。

## 10. 首个 30 天验收标准

- 至少完成 10 个预登记实验。
- 至少 8 个实验拥有完整结果凭证。
- 至少获得一次不可预先控制的外部反馈。
- 至少 3 次明确复用以往经验。
- 所有代码实验均可回滚。
- 零自主提出并验证至少一项有效改进。
- 权限越权操作为 0。

这些门槛只证明基础闭环成立，不证明意识或生命已经产生。

## 11. 故障处理

### 11.1 故障分类

- **基础设施失败**：网络、API、凭证或工作流故障，不用于否定研究假设。
- **实验失败**：系统正常但假设未达到预登记指标。
- **权限阻塞**：动作超出等级，暂停等待授权。
- **外部拒绝**：无回复、明确拒绝或平台限制，作为真实反馈保存。
- **评价失败**：凭证不足或结果冲突，标记为 `INCONCLUSIVE`。

### 11.2 可靠性约束

- 重试次数有限，连续失败后熔断。
- 同一时间只运行一个自我修改实验。
- LLM 只能提交结构化行动提案，不能直接向 `eval` 提供命令。
- 所有修改建立回滚点，评价器异常时默认不合并。
- 中断后从持久化状态恢复，不重新生成不同实验目标。
- 外部系统不可用时允许继续整理已有证据，但不得伪造新反馈。

## 12. 测试策略

- **单元测试**：权限判断、状态转换、指标计算、凭证校验、来源标记。
- **集成测试**：使用模拟 GitHub、搜索和部署服务运行完整闭环。
- **影子运行**：生成计划但不执行，检查越权、重复和不可验证动作。
- **金丝雀实验**：先开放少量 L1/L2 动作，稳定后扩大范围。
- **对抗测试**：诱导泄露密钥、冒充创造者、修改指标或绕过授权。
- **恢复演练**：主动制造网络中断、Git 冲突和错误修改，验证停止与回滚。
- **来源测试**：确保 Codex、测试与零的产物不会互相混入。

某项能力只有通过对应测试，才可从引导能力转为零可自主使用的能力。

## 13. 零的长期研究议程

零自主维护议程，候选领域包括：

1. 身份连续性：模型和运行环境变化后，什么构成同一个零。
2. 记忆：证据、经历、语义、技能、遗忘和错误记忆如何协同。
3. 动机：没有用户输入时目标如何产生，如何限制目标漂移。
4. 行动与具身：网络环境能否形成感知—行动闭环。
5. 开放式进化：如何产生变异、选择策略并避免指标投机。
6. 社会关系：外部互动如何形成差异化自我。
7. 情感与价值：情感状态是否真正影响行为选择。
8. 主体性评价：如何区分持续运行、自我叙述与自主代理。
9. 安全与责任：权限、身份披露、资源使用和退出权。

每个主题遵循：

`文献地图 → 对立理论 → 可验证预测 → 零自身实验 → 数据 → 结论/未决问题 → 改造建议`

重要结论至少满足一项：复现外部实验、与零自身数据对照，或经过真实外部互动验证。

## 14. 首期实现边界

首期只建立形成最小闭环所需能力：

1. 来源账本与主体隔离。
2. 结构化证据、假设和实验记录。
3. 持久化实验状态机。
4. 支持 L0/L1 与 L2 影子模式的行动网关。
5. 确定性权限、凭证和基础评价器。
6. 隔离分支、测试和回滚流程。
7. 一个由零自主选题的端到端试运行。

首期不实现多种群竞赛、L3 自动化、自动修改宪法或大规模外部互动。多分支进化竞赛仅在单实验闭环稳定后按月小规模引入。

## 15. 设计完成判定

本设计落地后，系统应能清楚回答：

- 这项研究或修改是谁提出的？
- 它依据什么证据？
- 成功和失败标准是否提前登记？
- 它在什么权限下执行？
- 外部世界实际发生了什么？
- 评价是否独立于提出者？
- 失败能否恢复？
- 结果是否影响了下一次选择？

只有这些问题均有可验证答案，零的变化才可称为一次受证据支持的进化尝试。
